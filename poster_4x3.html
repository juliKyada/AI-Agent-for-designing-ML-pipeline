Problem Domain & Motivation

  • Domain: Automated ML pipeline design for tabular datasets using metadata-driven constraints.
  • Manual pipeline selection is complex, slow, and dependent on expert trial-and-error.
  • Exhaustive AutoML evaluates many unsuitable models, increasing compute cost.
  • Our motivation is to prune the search space early using dataset characteristics.

Objective

  • Build MetaFlow, a rule-guided AutoML assistant based on dataset metadata.
  • Automatically detect task type and configure preprocessing + model families.
  • Rank model candidates with interpretable constraints and tunable grids.
  • Validate both performance quality and computational efficiency.

Proposed Architecture / Methodology

Input Data

Tabular dataset with target column.

Metadata Extraction

9 groups: schema, quality, outliers, cardinality, imbalance, and correlation.

Task Detection

Rule chain identifies classification or regression with confidence.

Pre-processing

Missing handling, cleaning, scaling, one-hot encoding.

Model Selection

Metadata-conditioned ranking and model-specific hyperparameter grids.

Pipeline + CV

Top-k pipelines tuned with cross-validation and evaluated on hold-out test set.

Best Output

Best model, issue flags, and optimization recommendations.

Dataset & Experimental Setup

Datasets

  • D1 Heart Disease: N=1,025, p=13, binary classification.
  • D2 Credit Card Fraud: N=284,807, p=30, severe class imbalance.
  • D3 Brain Cancer Gene Expression: N=130, p=54,676, multiclass.
  • D4 House Sales: N=21,613, p=19, regression.

Experimental Setup

  • 80/20 split (stratified for classification), grid-search CV for each candidate.
  • Top-k candidate pipelines selected via rule-based metadata filtering.
  • Classification: Accuracy, Precision, Recall, F1. Regression: R2, MSE, RMSE, MAE.

Key Observations

  • Metadata rules selected strong candidates across small, large, and high-dimensional datasets.
  • Imbalance and complexity modifiers improved practical model prioritization.
  • Candidate evaluation reduced from 9 to 3 in case studies (about 67% reduction).

Conclusion & Key Outcomes

  • RQ1: Lightweight metadata can guide accurate task and model selection.
  • RQ2: Rule-based pruning lowers compute without reducing quality.
  • MetaFlow balances interpretability, speed, and strong performance.
  • Future work: LLM-enhanced reasoning for richer metadata patterns.
