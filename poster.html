Research Problem & Motivation

  • Manual ML pipeline design is slow, iterative, and heavily expert-dependent.
  • Exhaustive AutoML wastes resources evaluating unsuitable model candidates.
  • A metadata-first strategy can prune search space before expensive training starts.
  • Goal: improve model selection efficiency without sacrificing final performance.

Objective

  • Build MetaFlow, a metadata-driven pipeline design framework.
  • Detect task type and route to suitable preprocessing and model families.
  • Prioritize candidates with interpretable rule-based constraints.
  • Validate efficacy and computational efficiency on real datasets.

Proposed Architecture / Methodology

Input Data

Tabular dataset with features and target.

Metadata Extraction

9 metadata groups: structure, quality, stats, outliers, imbalance, and more.

Task Detection

Rule chain infers classification vs regression with confidence.

Pre-processing

Missing value handling, row cleanup, scaling, and one-hot encoding.

Model Selection

Metadata-conditioned ranking of model families and hyperparameter grids.

Pipeline + CV

Top-k pipelines tuned with cross-validation and evaluated on hold-out test data.

Best Output

Best model, diagnostics, and optimization suggestions.

Dataset & Experimental Setup

Datasets Used

  • D1 Heart Disease: N=1,025, p=13, binary classification.
  • D2 Credit Card Fraud: N=284,807, p=30, extreme class imbalance.
  • D3 Brain Cancer Gene Expression: N=130, p=54,676, multiclass.
  • D4 House Sales: N=21,613, p=19, regression.

Setup

  • Top-k model candidates selected via metadata rules (default k=5).
  • Grid search with cross-validation on 80/20 train-test split.
  • Classification metrics: Accuracy, Precision, Recall, F1.
  • Regression metrics: R2, MSE, RMSE, MAE.

Key Observations

  • Metadata rules consistently selected high-performing model families.
  • High-dimensional and imbalanced data conditions were handled explicitly.
  • Search space reduced from 9 to 3 candidates in case studies.
  • Approximate reduction in model evaluations: 67%.
